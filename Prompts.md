# Prompts Used During Development

This document contains some of the prompts I used while building the Dev Detective project. Most of the prompts were focused on understanding concepts, debugging issues, and improving my implementation rather than directly generating solutions.

---

## Project Planning

- Help me understand the overall workflow of a GitHub profile finder application.
- Explain how this sprint should be approached step by step.
- Break the project into smaller milestones that I can complete one by one.

---

## HTML & CSS

- Review my HTML structure and suggest improvements.
- Help identify incorrect nesting of HTML elements.
- Explain why my layout is breaking.
- Suggest better ways to organize the profile card.
- Explain the difference between Flexbox and Grid for this layout.
- Review my CSS and suggest improvements for responsiveness.
- Help improve spacing, alignment, and overall UI consistency.

---

## JavaScript Concepts

- Explain Async/Await in simple language.
- Explain how the Fetch API works internally.
- Help me understand JSON responses from an API.
- Explain Promise.all() with a practical example.
- Explain how reduce() can be used to calculate total values.
- Explain why error handling is important while working with APIs.

---

## GitHub API

- Explain the GitHub REST API endpoints used in this project.
- Help me understand the structure of the GitHub user object.
- Explain how to fetch repositories after retrieving user data.
- Clarify the purpose of repos_url and how it is used.

---

## Debugging

- Help identify JavaScript errors.
- Explain why a particular API request was failing.
- Help debug HTML and CSS layout issues.
- Review my code and suggest cleaner approaches.
- Explain common mistakes while working with asynchronous JavaScript.

---

## Best Practices

- Suggest ways to make the UI more user-friendly.
- Recommend improvements without changing the overall project structure.
- Explain how to make the project more maintainable.
- Review the project against the sprint requirements and identify missing features.
